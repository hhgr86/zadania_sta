# Задание 1: Архитектура компании

---

## Часть 1. Соединение понятий

### 1. Правильная логическая цепочка
> **Стейкхолдеры** $\rightarrow$ **Сет ракурсов** $\rightarrow$ **Интересы**

### 2. Обоснование шагов цепочки
* **Стейкхолдеры:** Организация создается и функционирует для удовлетворения потребностей конкретных людей и групп. Именно стейкхолдеры являются первоисточником любых требований и целей.
* **Сет ракурсов:** Каждый стейкхолдер смотрит на компанию под определенным углом зрения.
* **Интересы:** Стейкхолдер формирует свои ожидания, цели и требования.

## Часть 3. Определения понятий

### a) Сет ракурсов
Представляет собой упорядоченную систему проекций и точек зрения для описания архитектуры организации. Его цель — разбить сложную систему на небольшие аспекты, снижая нагрузку при проектировании.
* **Связь с другими понятиями:** Напрямую соотносится со стейкхолдерами, так как отражает угол зрения определенной группы лиц. 
* **Практический пример:** Применяется при создании комплексных моделей компании, когда аналитик строит отдельные схемы для топ-менеджмента (бизнес-ракурс) и для разработчиков (технологический ракурс).

### b) Стейкхолдеры
Это любые индивидуумы, группы или организации, имеющие законные интересы, ожидания.
* **Связь с другими понятиями:** Тесно связаны с сетом ракурсов: каждая группа смотрит на систему через призму собственного ракурса.
* **Практический пример:** Работа начинается на этапе инициации проекта с составления карты заинтересованных сторон, проведения интервью для выявления их потребностей и согласования целей.

### c) Интересы
* **Суть и назначение:** Представляют собой потребности, ожидания, цели стейкхолдеров по отношению к организации. 
* **Связь с другими понятиями:** Непосредственно создаются стейкхолдерами через сет ракурсов.
* **Практический пример:** Применяются в сборе требований , где аналитик выявляет пользователей.
