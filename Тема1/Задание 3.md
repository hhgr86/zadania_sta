# Процесс: «Заказ товара онлайн» (Диаграмма BPMN с дорожками)

Ниже представлена структура диаграммы, стилизованная под формат на картинке (с разделением по ролям-дорожкам, шлюзами, сообщениями и уведомлениями), которую можно легко скопировать и вставить на GitHub:

```mermaid
graph TD
    classDef poolStyle fill:#ffffff,stroke:#333,stroke-width:1px;

    subgraph Клиент
        Start([Запрос на товар]) 
        MakeOrder[Оформить заказ онлайн]
        ReceiveNotice[Получение товара]
        RateService[Оценка сервиса]
    end

    subgraph Менеджер интернет-магазина
        GetOrder[Получение заказа]
        CreateTask[Создать заявку для склада]
        SendNoticeNode((Уведомление клиенту))
        IssueGoods[Выдача товара клиенту]
        CloseTask1[Закрыть заявку в Системе №1]
        CloseTask2[Закрыть заявку в Системе №2]
        End([Процесс завершен])
    end

    subgraph Склад
        ReceiveTask[Получение заявки в системе]
        CheckStock[Проверить наличие товара]
        StockCond{Товар в наличии?}
        CallLogistics[Вызвать логистику]
        SendToManager[Отправить товар менеджеру]
    end

    subgraph Логистика
        Delivery[Доставка товара]
    end

    subgraph Поставщик
        WaitSupplier[Ожидание оборудования]
        DeliverSupplier[Поставщик доставляет]
    end

    %% Логика процесса по шагам
    Start --> MakeOrder
    MakeOrder -.->|Сообщение| GetOrder
    
    GetOrder --> CreateTask
    CreateTask -.->|Сообщение| ReceiveTask
    
    ReceiveTask --> CheckStock
    CheckStock --> StockCond
    
    StockCond -- Да --> CallLogistics
    CallLogistics --> SendToManager
    
    StockCond -- Нет --> WaitSupplier
    WaitSupplier --> DeliverSupplier
    DeliverSupplier --> Delivery
    
    SendToManager -.->|Сообщение| IssueGoods
    Delivery --> IssueGoods
    
    IssueGoods --> CloseTask1
    CloseTask1 --> CloseTask2
    CloseTask2 --> End

    SendToManager -.->|Уведомление| SendNoticeNode
    SendNoticeNode -.->|Email / SMS| ReceiveNotice
    ReceiveNotice --> RateService
