# Процесс: «Заказ товара онлайн» (Диаграмма BPMN с ролями и элементами)

Ниже представлена графическая BPMN-диаграмма процесса, адаптированная для корректного отображения на GitHub (через Mermaid):

```mermaid
graph TD
    subgraph Клиент
        Start([Старт: Заказ оформлен])
        ReceiveNotice[Получить уведомление об отправке]
        End([Конец процесса])
    end

    subgraph Менеджер интернет-магазина
        GetOrder[Получение заказа]
        CheckStock[Проверка наличия товара]
        StockCond{Товар есть<br>в наличии?}
        TimerWait((Таймер:<br>Ожидание поставщика))
    end

    subgraph Бухгалтерия
        PrepDocs[Подготовка документов<br>и накладных]
    end

    subgraph Работник склада
        PackGoods[Комплектация и<br>упаковка товара]
    end

    subgraph Служба доставки / Логист
        ShipGoods[Отправка товара<br>клиенту]
    end

    %% Логика процесса
    Start --> GetOrder
    GetOrder --> CheckStock
    CheckStock --> StockCond
    
    %% Условное ветвление и цикл
    StockCond -- Нет --> TimerWait
    TimerWait --> CheckStock
    
    %% Параллельное разделение (AND-gateway)
    StockCond -- Да --> Split(( ))
    Split --> PrepDocs
    Split --> PackGoods
    
    %% Параллельное слияние (AND-gateway)
    PrepDocs --> Join(( ))
    PackGoods --> Join
    
    %% Последовательность к отправке
    Join --> ShipGoods
    
    %% Информационный узел / Уведомление
    ShipGoods --> SendNotice[Информационный узел:<br>Отправка SMS / Email]
    SendNotice --> ReceiveNotice
    ReceiveNotice --> End
